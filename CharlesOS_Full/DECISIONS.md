# Decisions

## Format

Use this structure for each important decision:

### YYYY-MM-DD: Decision title

Decision:
What was decided.

Reason:
Why this was chosen.

Alternatives considered:
Other options.

Rejected because:
Why the alternatives were not chosen.

Impact:
What this changes.

---

## 2026-06-06: GitHub as CharlesOS source of truth

Decision:
Use GitHub as the source of truth for CharlesOS.

Reason:
GitHub works well with Cursor, Claude, Codex, Lovable and project-based workflows.

Alternatives considered:
Notion, Apple Notes, local folders only.

Rejected because:
They are less useful inside AI coding and product-building tools.

Impact:
All markdown files should be maintained in GitHub first.
