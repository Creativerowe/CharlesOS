# AI Stack

## Purpose

This file defines Charles' AI-enabled product creation workflow.

## Core Role of Each Tool

### ChatGPT

Use for:

- Strategy
- Thinking
- Critique
- Writing
- Frameworks
- Planning
- Portfolio
- Coaching
- Prompt design

### Claude

Use for:

- Long-form reasoning
- Technical planning
- Code explanation
- Complex product thinking
- Structured documentation

### Cursor

Use for:

- Codebase work
- Refactoring
- Debugging
- Multi-file edits
- Using agents inside a real project

### Codex

Use for:

- Coding support
- Reviewing code
- Generating implementation options
- Solving technical problems

### Lovable

Use for:

- Fast app prototypes
- Full-stack MVPs
- Early product exploration
- Turning product briefs into working products

### Figma

Use for:

- Interface design
- Design systems
- Prototyping
- Visual exploration

### Figma Make / Figma Agent

Use for:

- Design generation
- Layout exploration
- Rapid UI ideation

### Weavy

Use for:

- Node-based image workflows
- Consistent visual asset generation
- Campaign visuals
- Reusable creative systems

### Midjourney

Use for:

- High-quality visual exploration
- Moodboards
- Campaign-style imagery
- Reference image creation

### GitHub

Use for:

- Source of truth
- Version control
- Documentation
- Collaboration
- AI-readable project history

## Working Principle

Do not rely on chat memory alone.

Use markdown files as the source of truth.

## Typical Workflow

1. Define brief in ChatGPT
2. Store context in markdown
3. Explore visuals in Figma / Weavy / Midjourney
4. Prototype in Lovable
5. Move serious build work to Cursor
6. Commit to GitHub
7. Update CharlesOS with decisions and learnings
